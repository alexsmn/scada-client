Руководство пользователя

Установка сервера:

  1. Создать папку Server и скопировать туда Server.exe.
  2. Запустить Server.exe. Следует разрешить брандмауэру открыть порт при запросе.

На экране появится черное окно, сервер создаст новую конфигурацию, подпапку Logs (логи), файлы таблиц (с расширением tbl), файлы архивов (с расширением arc), после чего сервер будет готов к работе.
Завершение работы сервера выполняется нажатием Alt+F4. Закрытие крестиком в заголовке окна приведет к ошибке.
Внимание: при первом запуске сервер пересоздает конфигурацию. Для того чтобы пересоздать тестовую конфигурацию при последующих запусках следует запустить Server.exe с параметром /reinit.

Установка клиента:

1. На том же компьютере создать папку Client и скопировать туда Workplace.exe, page.xml, welcome.htm.
2. Запустить Workplace.exe. Будет создана подпапка Logs. Клиент готов.

Работа с клиентом

Сейчас клиент не ограничен правами пользователя. Но в дальнейшем предусматривается два режима работы - пользовательский и администраторский. В пользовательском режиме разрешается выполнять функции необходимые для работы оператора, в администраторском дополнительно разрешено редактировать конфигурацию сервера. Все конфигурация хранится на сервере в специализированных таблицах.

1. Регистрация

В поле Имя вести имя пользователя (предопределены Клиент 1 - Клиент 10) и нажать ОК. Кнопка ОК разрешена только при наличии соединения с сервером.

2. Данные

Кнопка Данные открывает таблицу сервера и доступна в только режиме администрирования. Любая таблица может быть изменена. На данном этапе делать это нужно аккуратно, т.к. неверные значения полей в некоторых случая могут привести к краху системы (все случаи еще предстоит установить, но при правильном конфигурировании это не критично).

При нажатии кнопки Данные открывается контекстное меню со списком всех таблиц сервера.

  • TS - объекты ТС;
  • TIT - объекты ТИТ;
  • TII - объекты ТИИ;
  • GROUP - группы объектов;
  • USER - пользователи, подсистемы и их права;
  • EVENT - текущие (неквитированные) события;
  • SUBS - подсистемы;
  • EXPR - дорасчеты;
  • IEC_CHAN - МЭК: каналы;
  • IEC_DEV - МЭК: устройства (не используется);
  • IEC_ITEM - МЭК: объекты.
При выборе одного из пунктов меню открывается таблица. Структура каждой таблицы отличается, но все из них имеют поле ID, значение которого уникально в пределах этой таблицы. Это значение не может быть изменено и при создании записи генерируется автоматически.

Изменение данных таблицы выполняется двойным щелчком в ячейке, или выбором ячейки и последующим вводом значения. После ввода значения, подтвердите его нажатием Enter. Значение запишется в ячейку, а измененная запись будет выделена желтым цветом, обозначающим режим редактирования. Пока запись находится в этом режиме, введенные данные не отправляются на сервер, но отображаются на экране. Для отправки данных на сервер следует перейти на другую запись или повторно нажать Enter (второй способ бывает востребован при добавлении новой записи - см. далее). После чего желтое выделение будет убрано, и запись временно примет исходное состояние. После обработки данных сервером, запись будет обновлена в соответствии с ними.

Последняя запись таблицы, заполненная "...", используется для создания новой записи. Для этого нужно ввести значения в ячейки этой записи, и нажать Enter.

Поля таблиц TS, TIT, TII, GROUP:
  • PAR - общее для таблиц GROUP, TS, TIT и TII, определяет ID родительской группы в таблице GROUP. Значение PAR равное нулю определяет нулевой уровень иерархии;
  • NAME - определяет диспетчерское наименование объекта (не более 50 символов, уникальность не требуется).
  • VAL - значение объекта после преобразования;
  • RAW - значение объекта полученное с устройства (до преобразования);
  • VAL_TM - время изменения значения;
  • VAL_UID - ID пользователя или подсистемы в таблице UID, причастных к изменению значения объекта;
  • QUAL - достоверность (Да/Нет);
  • QUAL_TM - время изменения достоверности;
  • QUAL_UID - ID пользователя или подсистемы в таблице UID, причастных к изменению достоверности объекта;
  • INV - инверсия значения ТС;
  • LBL_OPEN - метка состояния "Откл" (контакт разомкнут);
  • LBL_CLOSE - метка состояния "Вкл" (контакт замкнут);
  • CONV - способ преобразования объекта (сейчас используется линейное преобразование);
  • CLAMP - ограничение значения объекта его диапазоном: 0 - нет, 1 - EU_LO/EU_HI (сейчас не используется);
  • EU_LO - нижняя граница диапазона значения после преобразования;
  • EU_HI - верхняя граница диапазона значения после преобразования;
  • IR_LO - нижняя граница физического диапазона (значения до преобразования);
  • IR_HI - верхняя граница физического диапазона (значения до преобразования).

Поля таблицы EVENT:

  • TIME - время события;
  • TYP - тип события: 0 - требует, 1 - не требует квитирования (сейчас всегда 1);
  • TID - номер таблицы объекта инициировавшего событие. Для событий не связанных с объектом равно нулю;
  • RID - соответствующий номер объекта в таблице TID;
  • UID - номер в таблице USER пользователя или подсистемы, инициировавшей событие;
  • MSG - текстовое описание события.

Таблица EXPR определяет список дорасчетных объектов ТС, ТИТ, ТИИ, устанавливает в соответствие каждому из них некоторое выражение. Поля таблицы:

  • TID - номер таблицы дорасчетного объекта;
  • RID - номер дорасчетного объекта в таблице TID;
  • EXPR - выражение (не длиннее 200 символов). В выражении могут использоваться арифметические и логические операции, функции Abs, Sin, Cos, Tan, ASin, ACos, ATan, Sqrt, а также переменные вида вида TS.1 (ТС с номером 1), TIT.23, TII.34.

Таблицы МЭК устанавливают связь между каналами сервера МЭК и объектами сервера. Для этого следует перечислить сервера МЭК в таблице IEC_CHAN, и заполнить диапазоны объектов в таблице IEC_ITEM. Поля таблиц протокола МЭК:

  • HOST - адрес IP или имя компьютера сервера МЭК-104;
  • PAR - номер канала МЭК в таблице IEC_CHAN;
  • ADDR - адрес первого объекта МЭК;
  • COUNT - количество объектов МЭК с подряд-идущими адресами;
  • TID - номер таблицы, в которой находятся связанные объекты;
  • RID - номер первого объекта в таблице TID.

3. График

При нажатии кнопки График открывается окно с графиком значений первого объекта ТИТ за последний час. Любое последующее изменение значения объекта ТИТ также фиксируется на графике, после чего график смещается.

Прокрутка графика вперед и назад во времени осуществляется щелчком в области графика и перемещением мыши влево-вправо. При этом невидимые ранее области будут автоматически заполнены данными при их наличии.

Щелчок по горизонтальной шкале и перемещение мыши приведет к изменению масштаба времени.

4. Кадр

Реализовано частично.

5. Объекты

При нажатии кнопки Объекты отображается дерево объектов системы, упорядоченных по группам. Щелкнув правой клавишей по объекту ТИТ и выбрав в контекстном меню График, можно отобразить график по этому объекту.

6. Таблица

Создает пустую таблицу для наблюдения данных оператором. Для добавления объекта в таблицу нужно щелкнуть в поле Имя любой строки и ввести путь объекта ТС в формате "таблица.запись". Реализовано частично.

7. Пользовательская таблица

Создает пользовательскую таблицу, в каждую ячейку которой можно вывести значение одного из объектов. 8. Сводка Отображает ретроспективу по объектам ТС и ТИТ. Реализовано частично.

9. События

Реализовано частично. После квитирования событие удаляется из таблицы событий и сохраняется в архиве.

Разрыв связи

При разрыве связи с сервером клиент пытается восстановить соединение. Если сервер отклонил попытку восстановления, клиент предлагает ввсети имя пользователя и пароль.

Настройка нового канала МЭК-104

1. Добавить новую запись в таблицу IEC_CHAN:

  • NAME - имя нового канала;
  • HOST - адрес IP или имя хоста сервера МЭК-104.

2. Добавить новую запись в таблицу IEC_ITEM:

  • PAR - номер канала добавленного на предыдущем шаге;
  • ADDR - адрес первого обрабатываемого объекта МЭК;
  • COUNT - количество объектов МЭК с последовательно идущими адресам;
  • TID - номер таблицы: 1 - ТС, 2 - ТИТ, для хранения объектов;
  • RID - номер первой записи в таблице TID.

Ошибки

  • Клиент - Модальный диалог провоцирует повторный вход в MsgSock::OnRecv.
  • Клиент - Всплывающая панель инструментов редактирования пользовательской таблицы.
  • Клиент - Польз таблица - Бордюры и цвета ячеек.
  • Сервер - Архивирование срезов.
  • Сервер - Параметры сервера - Количество события для автоквитирования.
  • Сервер - Параметры сервера - Параметры архивов.
